## 🔴 Bugs

- [x] **Top 5 Items bar chart X-axis shows 0–1 instead of real quantities**
  - Location: `resources/js/pages/dashboard.tsx` — Recharts BarChart for `topItems`
  - Cause: Missing or wrong `domain` / `tickFormatter` on XAxis
  - Fix: Set `<XAxis type="number" allowDecimals={false} />` and remove any normalization in data mapping
  > Fixed in: commit 0fedb64 — added `allowDecimals={false} domain={[0, 'dataMax']}` to XAxis

- [x] **Items/Produk — 404 in production build**
  - Location: `/items` route
  - Cause: Stale Vite manifest when running prod build without rebuilding
  - Fix: Always run `npm run build` before testing. Confirmed working in dev server.
  > Fixed in: was a process issue (stale build), resolved by `npm run build` on 2026-03-10

## 🟠 High Priority UX

- [x] **POS Terminal item cards too small for touch use**
  - Location: `resources/js/pages/pos/Terminal.tsx` — item grid section
  - Issue: Cards are very small at 1440px; on a real POS touchscreen (1024px) they'd be unusable
  - Suggestion: Increase card min-size, larger font for name/price, add grid density toggle (large/small/list)
  > Fixed: reduced max columns to xl:4, increased padding to p-3.5, larger fonts (text-base), added grid/compact density toggle

- [x] **Users page — 5 unlabeled action icons**
  - Location: `resources/js/pages/Users/Index.tsx` — action column
  - Icons: pencil (edit), shield (permissions), building (warehouses), key (reset password), trash (delete)
  - Issue: No labels, no tooltips. Unusable on touch devices.
  > Already implemented — all 5 buttons have Tooltip components with Indonesian labels. Not a real issue.

- [x] **Empty states have no guidance or CTAs**
  - Affected pages: Supplier, Customer, Purchase Order, Returns, Stock Transfer, Stock Adjustment
  - Issue: All show generic "Tidak ada X ditemukan" with no next-step guidance
  > Fixed: all 6 pages now show icon + heading + description + CTA button (commit after Task B)

- [x] **Warehouse detail page uses dark theme — inconsistent with rest of app**
  - Location: `resources/js/pages/warehouse/Show.tsx`
  - Issue: Dark background table while every other page is light-themed
  > Not a real issue — code uses theme-aware classes (bg-muted). Dark appearance was OS dark mode in screenshots.

- [x] **Minimum stock is 0 everywhere — stock alert system never fires**
  - Issue: All items have `stok_minimal = 0`, making the Dashboard "Stok Minim" KPI and alert table useless
  > Fixed: added `itemsWithNoMinimum` count to DashboardController + amber banner on dashboard when count > 0 (commit e9dd7fa)

## 🟢 Feature Suggestions

- [ ] **POS Terminal — auto-focus search on any keypress (barcode scanner UX)**
  - Location: `resources/js/pages/pos/Terminal.tsx`
  - Suggestion: `document.addEventListener('keydown', ...)` to auto-focus the item search input
  - Benefit: Barcode scanners send keypresses; cashier doesn't need to click search first

- [ ] **POS Terminal — receipt printing / WhatsApp receipt after checkout**
  - Location: `resources/js/pages/pos/Terminal.tsx` — post-checkout flow
  - Suggestion: After successful sale, show modal with options:
    - Print receipt (browser `window.print()` with receipt CSS)
    - Send via WhatsApp (`wa.me/?text=...` with transaction summary)

- [ ] **Dashboard — add date range selector**
  - Location: `resources/js/pages/dashboard.tsx`
  - Issue: Dashboard always shows "today" and "this month" — no way to review other periods
  - Suggestion: Add quick selector: Hari Ini / Minggu Ini / Bulan Ini / Tahun Ini / Custom

- [ ] **Items — per-item stock movement history**
  - Issue: No way to see an item's transfer/adjustment/sale history from the item detail page
  - Suggestion: Add "Riwayat Stok" tab on item detail showing chronological stock events

- [ ] **Purchase Order — quick-add supplier from PO creation form**
  - Issue: PO requires a supplier, but adding a supplier requires leaving the PO form
  - Suggestion: Add "+ Supplier Baru" inline option inside the supplier dropdown on PO form

- [ ] **Verify Laporan Stok and Laporan Kas are fully implemented**
  - Sidebar shows: Laporan Stok, Laporan Penjualan, Laporan Kas, Laba Rugi
  - Laporan Penjualan ✅ and Laba Rugi ✅ confirmed working
  - Laporan Stok and Laporan Kas — check if pages are built and functional

- [ ] **Add seeder with realistic demo data**
  - Run: `php artisan db:seed` currently gives almost no data to work with
  - Suggestion: Create `DemoDataSeeder` with:
    - 10 suppliers, 20 customers
    - 5 purchase orders (various statuses)
    - 30 days of sales transactions
    - Stock transfers between warehouses
    - Realistic `stok_minimal` values on items

# Hash ID Obfuscation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace all raw integer database IDs exposed to the browser with short, reversible hash strings so sequential IDs cannot be enumerated or used to infer business metrics.

**Architecture:** Install `hashids/hashids` (pure PHP, no Laravel wrapper needed). A static `HashId` helper wraps it. Two global functions `hid(int): string` and `dhid(string): int` are available everywhere. A `HasHashId` Eloquent trait overrides `getRouteKey()` / `resolveRouteBinding()` on every route-bound model so Laravel's route model binding transparently handles encoding/decoding. Controllers encode all outgoing IDs with `hid()` and decode any incoming foreign-key IDs (e.g., `warehouse_id`, `customer_id` in POST bodies) with `dhid()`. The frontend receives strings; it sends them back unchanged; the DB never stores or sees hashes. The salt is stored in a dedicated `HASH_ID_SALT` env variable — never derived from `APP_KEY` (which rotates independently).

**Tech Stack:** PHP 8.2, Laravel 12, `hashids/hashids ^5.0`, Inertia.js v2, React 19, TypeScript, Dexie (IndexedDB for offline POS cart).

## Encoding rules (apply consistently everywhere)

| Scenario | PHP | TypeScript |
|---|---|---|
| Sending id to frontend | `'id' => hid($model->id)` | receive as `string` |
| Sending FK to frontend | `'warehouse_id' => hid($fk)` | receive as `string` |
| Receiving FK from request | `$warehouseId = dhid($request->warehouse_id)` | send back unchanged |
| Route model binding | automatic via `HasHashId` trait | `route('x.show', model.id)` works unchanged |
| `dhid(null\|'')` returns | `0` (safe; findOrFail(0) → 404) | — |

- [ ] **Step 1.2 — Create `app/Helpers/HashId.php`**

```php
<?php

namespace App\Helpers;

use Hashids\Hashids;

class HashId
{
    private static ?Hashids $instance = null;

    private static function instance(): Hashids
    {
        if (self::$instance === null) {
            self::$instance = new Hashids(
                salt: config('app.hash_id_salt', 'fallback-salt-change-me'),
                minHashLength: 8,
                alphabet: 'abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ1234567890'
            );
        }

        return self::$instance;
    }

    public static function encode(int $id): string
    {
        return self::instance()->encode($id);
    }

    /**
     * Returns 0 (not null) on invalid input so findOrFail(0) triggers a clean 404.
     */
    public static function decode(string $hash): int
    {
        if ($hash === '' || $hash === '0') {
            return 0;
        }

        $decoded = self::instance()->decode($hash);

        return isset($decoded[0]) ? (int) $decoded[0] : 0;
    }
}
```

- [ ] **Step 1.3 — Create `app/Helpers/helpers.php`**

```php
<?php

use App\Helpers\HashId;

if (! function_exists('hid')) {
    /**
     * Encode an integer database ID to a URL-safe hash string.
     * Returns '' if $id is null or 0.
     */
    function hid(int|null $id): string
    {
        if ($id === null || $id === 0) {
            return '';
        }

        return HashId::encode($id);
    }
}

if (! function_exists('dhid')) {
    /**
     * Decode a hash string back to an integer ID.
     * Returns 0 on invalid/empty input (so findOrFail(0) gives a 404).
     */
    function dhid(string|null $hash): int
    {
        if ($hash === null || $hash === '') {
            return 0;
        }

        return HashId::decode($hash);
    }
}
```

## Task 2: HasHashId Eloquent trait

**Files:**
- Create: `app/Traits/HasHashId.php`

- [ ] **Step 2.1 — Create `app/Traits/HasHashId.php`**

```php
<?php

namespace App\Traits;

use App\Helpers\HashId;

trait HasHashId
{
    /**
     * Return the hash-encoded primary key for URL generation.
     * Laravel uses this when building routes (e.g. route('item.show', $item)).
     */
    public function getRouteKey(): string
    {
        return hid((int) $this->getKey());
    }

    /**
     * Decode the incoming hash from the URL and resolve the model by integer PK.
     * Uses ?static (not ?self) so PHP resolves to the concrete model class.
     */
    public function resolveRouteBinding(mixed $value, $field = null): ?static
    {
        $id = is_string($value) ? HashId::decode($value) : (int) $value;

        if ($id === 0) {
            return null; // triggers 404
        }

        /** @var static|null */
        return $this->where($field ?? $this->getRouteKeyName(), $id)->first();
    }

    /**
     * Ensure the route key column stays as the real primary key column.
     */
    public function getRouteKeyName(): string
    {
        return $this->getKeyName(); // 'id'
    }
}
```

class PosDatabase extends Dexie {
    cart!: Table<SavedCart, number>;
    pendingTransactions!: Table<PendingTransaction, number>;

    constructor() {
        super('PosOfflineDB');
        this.version(1).stores({
            cart: '++id',
            pendingTransactions: '++id, status, idempotencyKey',
        });
        // Version 2: clears cart (numeric IDs incompatible with hash IDs).
        // ⚠️ DEPLOY NOTE: also clears pendingTransactions to prevent
        // numeric-ID payloads from being submitted post-deploy (they would 404).
        // Operators MUST ensure all pending offline transactions are synced
        // before deploying this change.
        this.version(2).stores({
            cart: '++id',
            pendingTransactions: '++id, status, idempotencyKey',

            tx.table('cart').clear();
            return tx.table('pendingTransactions').clear();

## Common pitfalls

| Pitfall | Fix |
|---|---|
| `dhid(null)` throws | `dhid()` accepts `string\|null` and returns 0 on null/empty |
| `findOrFail(0)` → 500 instead of 404 | `resolveRouteBinding` returns `null` (not throws) for hash decode → 0; Laravel turns null into 404 |
| Offline cart has stale numeric IDs after upgrade | IndexedDB version 2 clears both `cart` and `pendingTransactions` |
| `PosController::items()` returns wrong outlet prices | Change `(int) $request->get('warehouse_id')` → `dhid(...)` and key response by `hid($item->id)` |
| ReportController warehouse filter silently returns zero | Change every `(int) $request->get('warehouse_id')` → `dhid(...)` in all 8+ report methods |
| `outletPrices` keyed by `Number(hash)` → NaN | Retype as `Record<string, number>`; remove `Number(k)` cast |
| Promotion `appliesId` sent as `''` not `null` | `hid(null)` returns `''`; frontend checks `!promo.appliesId` instead of `promo.appliesId === null` |
| `auth.user.id` leaks as integer via Inertia share | Override in `HandleInertiaRequests::share()` — covered in Task 4b.5 |
| `allowedWarehouseIds` leaks integers via Inertia share | Encode with `->map(fn($id) => hid($id))` — covered in Task 4b.5 |
| `?self` return type on trait method → PHP TypeError | Use `?static` — already corrected in Task 2 code |
| Rotating `HASH_ID_SALT` / `APP_KEY` invalidates all URLs | Use a dedicated `HASH_ID_SALT` env var; never rotate it |
| `parseInt(selectValue)` in Terminal.tsx → NaN | Remove all `parseInt()` / `Number()` casts on hash ID values |
